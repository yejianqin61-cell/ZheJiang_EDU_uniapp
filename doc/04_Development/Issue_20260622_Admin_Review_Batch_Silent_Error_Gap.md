# Issue: 管理端入库审核页仍存在批量静默失败分支

日期：2026-06-22

## 现象

- `frontend-web/src/pages/admin/review/index.vue` 在批量审核操作中仍使用空 `catch`
- 当管理员取消确认框或批量审核接口失败时，页面没有区分用户取消与真实失败，真实失败也不会获得明确反馈

## 风险

- 入库审核属于后台关键运营链路，静默失败会让管理员误判批量操作是否执行成功
- 批量审核失败如果没有页面内提示，后续题目状态与后台预期不一致时排查成本会明显上升

## 处理建议

1. 为批量审核区分确认框取消与真实接口失败
2. 对真实失败补充明确错误提示
3. 增加页面回归测试，覆盖批量取消和批量失败分支
