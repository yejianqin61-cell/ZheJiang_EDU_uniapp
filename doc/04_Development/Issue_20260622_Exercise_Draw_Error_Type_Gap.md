# Issue: 练习抽题页错误解析仍在使用弱错误类型

日期：2026-06-22

## 现象

- `frontend-web/src/pages/exercises/draw.vue` 的抽题异常分支仍在使用 `catch (e: any)`
- `resolveDrawErrorMessage()` 直接消费 `error: any`，错误结构没有显式约束

## 风险

- 练习抽题链路的异常对象继续绕开类型系统。
- 后续如果扩展抽题失败分支或后端错误结构，弱类型解析容易掩盖空值和分支遗漏。

## 处理建议

1. 使用 `unknown` 接住抽题异常
2. 为错误解析收口最小结构类型
3. 保持现有“暂无试卷 / 网络失败 / 服务异常”提示语义不变
