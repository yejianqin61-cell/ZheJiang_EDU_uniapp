# Issue: 支付页余额加载链路仍存在静默失败

日期：2026-06-22

## 现象

- `frontend-web/src/pages/payment/index.vue` 在初始化账户余额时仍使用空 `catch`
- 当余额接口失败时，页面会静默展示为 `0`，用户无法判断是接口异常还是余额确实不足

## 风险

- 支付页属于下单后的关键资金链路，静默失败会误导用户对可支付能力的判断
- 用户可能在接口异常时被错误引导到其他支付方式，增加支付链路排查成本

## 处理建议

1. 为余额加载失败补充明确错误提示
2. 保留默认值兜底，避免支付页渲染中断
3. 增加页面回归测试，覆盖余额初始化失败时的提示行为
