# Issue: 管理端题库管理页错误分支反馈不完整

## 问题描述

- `frontend-web/src/pages/admin/questions/index.vue` 在题库列表加载失败时直接吞掉异常
- 删除题目时把确认框取消和真实接口失败放在同一个空 `catch` 里，真实失败会被静默吞掉

## 影响范围

- Web 端管理后台题库管理页
- 题库列表加载
- 题目删除操作

## 建议处理

- 为列表加载和删除失败补充统一错误消息提取逻辑
- 在列表加载失败时清空当前列表，避免旧态残留
- 区分确认框取消与真实接口失败，仅对真实失败提示用户
- 补充页面回归测试，覆盖列表失败和删除失败分支
