# Issue: 管理端提现页异常处理仍有弱类型与静默失败缺口

日期：2026-06-22

## 现象

- `frontend-web/src/pages/admin/withdrawals/index.vue` 的通过操作仍在使用 `catch (error: any)`
- 拒绝提现分支把弹窗取消和接口失败都落在同一个空 `catch` 中，真实失败不会给出提示

## 风险

- 管理端提现审核异常继续绕开类型系统。
- 拒绝提现请求失败时页面静默无反馈，运营侧难以及时判断是否真正处理成功。

## 处理建议

1. 使用 `unknown` 接住审核异常
2. 区分弹窗取消与接口失败，只对取消保持静默
3. 保持原有列表刷新和审核提示行为不变
