# Issue: 管理端订单页状态更新异常仍在使用弱错误类型

日期：2026-06-22

## 现象

- `frontend-web/src/pages/admin/orders/index.vue` 的状态更新分支仍在使用 `catch (error: any)`
- 页面已经复用共享 `admin` API 模块，但异常分支没有同步收口

## 风险

- 管理端订单状态流转异常继续绕开类型系统。
- 后续如果扩展物流节点、批量操作或错误提示，弱类型分支会继续扩散。

## 处理建议

1. 使用 `unknown` 接住状态更新异常
2. 在页面内收口最小错误消息提取逻辑
3. 保持原有列表刷新和状态提示行为不变
