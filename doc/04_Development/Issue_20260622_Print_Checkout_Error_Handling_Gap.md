# Issue: 打印结算页下单异常仍有弱类型与静默失败缺口

日期：2026-06-22

## 现象

- `frontend-web/src/pages/print/checkout/index.vue` 的下单分支仍在使用 `catch (error: any)`
- 页面只在异常对象缺少 `response` 时提示失败，服务端返回业务错误时会静默

## 风险

- 打印下单链路的异常对象继续绕开类型系统。
- 用户在下单失败时拿不到后端业务错误提示，容易重复提交或误判订单状态。

## 处理建议

1. 使用 `unknown` 接住下单异常
2. 在页面内收口最小错误消息提取逻辑
3. 区分网络失败兜底提示与服务端业务错误提示
4. 保持原有下单成功跳转与价格展示行为不变
