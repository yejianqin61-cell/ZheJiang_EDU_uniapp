# Issue: 订单详情页加载与导出失败兜底提示仍不稳定

日期：2026-06-22

## 现象

- `frontend-web/src/pages/orders/detail/index.vue` 在订单加载失败和导出失败时仍使用固定提示，缺少统一错误消息提取逻辑
- 当错误对象不符合标准 `Error` 结构时，页面提示行为没有明确测试约束

## 风险

- 订单详情页承载下载与导出关键动作，错误提示不稳定会影响用户判断和问题定位
- 后端错误结构变化时，如果没有统一兜底，前端容易出现不一致提示或信息量不足

## 处理建议

1. 统一详情页加载与导出失败的错误消息提取逻辑
2. 增加无 `message` 错误对象时的兜底提示测试
3. 与订单列表页一起形成订单链路错误处理回归
