# Issue: 管理端题目详情页删除动作仍在使用弱错误类型

日期：2026-06-22

## 现象

- `frontend-web/src/pages/admin/questions/detail/index.vue` 中删除题目动作仍使用 `catch (error: any)`
- 页面已经通过共享 `admin` API 模块获取题目详情并执行删除，但错误分支没有同步收口

## 风险

- 删除动作异常对象继续绕开类型系统。
- 管理端题库链路仍残留孤立的 `any`，容易在后续维护时继续扩散。

## 处理建议

1. 使用 `unknown` 接住删除动作异常
2. 在页面内收口最小错误消息提取逻辑
3. 保持原有删除成功/失败提示和返回行为不变
