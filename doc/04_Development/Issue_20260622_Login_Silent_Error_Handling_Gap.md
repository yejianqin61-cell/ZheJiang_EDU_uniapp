# Issue: 登录页多条认证链路仍存在静默失败

日期：2026-06-22

## 现象

- `frontend-web/src/pages/login/index.vue` 中短信发码、短信登录、邮箱发码、邮箱注册、邮箱登录、Dev 登录多个分支仍在使用空 `catch`
- 请求失败时页面缺少明确反馈，用户只能停留在当前表单而不知道失败原因

## 风险

- 登录与注册主链路失败时前端静默，影响可用性与排障效率。
- 认证页后续再扩展风控、校验和多角色入口时，静默失败分支容易继续扩散。

## 处理建议

1. 为登录页统一收口最小错误消息提取逻辑
2. 将空 `catch` 改为显式失败提示
3. 保持原有校验、成功提示、倒计时和跳转行为不变
