# Issue: 订单列表页下载失败兜底提示仍不稳定

日期：2026-06-22

## 现象

- `frontend-web/src/pages/orders/index.vue` 在获取下载链接失败时只给固定提示，没有统一错误消息提取逻辑
- 当错误对象缺少标准 `Error` 结构时，页面提示行为缺少明确测试约束

## 风险

- 订单列表页是用户重复访问的高频入口，下载失败提示不稳定会降低问题定位效率
- 后端错误结构变化时，如果前端没有稳定兜底，用户容易遇到无信息量或不一致提示

## 处理建议

1. 统一下载失败错误消息提取逻辑
2. 增加无 `message` 错误对象时的兜底提示测试
3. 顺手收口该页测试中的弱类型断言
