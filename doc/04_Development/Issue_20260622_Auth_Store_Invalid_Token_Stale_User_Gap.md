# Issue: 认证 store 在无效 token 场景下可能残留旧用户态

日期：2026-06-22

## 现象

- `frontend-web/src/stores/auth.ts` 在解析无效 token 时只清理 token 和本地存储，没有显式清理 `user`
- 当 store 已经持有旧用户信息时，无效 token 分支可能让界面保留过期登录态

## 风险

- 登录态恢复属于全站基础链路，残留旧用户态会影响权限判断和页面展示
- token 损坏或过期后的前端表现如果不一致，定位登录态异常会更困难

## 处理建议

1. 抽出统一的认证清理逻辑，同时清理 token、user 和本地存储
2. 在 token 缺失或无效时显式回到未登录状态
3. 增加 store 回归测试，覆盖无效 token 初始化分支
