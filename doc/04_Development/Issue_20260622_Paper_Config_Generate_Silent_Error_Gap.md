# Issue: 组卷配置页生成失败仍会静默停留

日期：2026-06-22

## 现象

- `frontend-web/src/pages/paper/config/index.vue` 在调用 `paper.generate()` 失败时仍使用空 `catch`
- 用户点击“开始 AI 组卷”后，如果组卷接口失败，页面只会停留在当前配置页，不会给出明确错误提示

## 风险

- 组卷入口属于核心主链路，静默失败会让用户误判为“仍在生成中”或“页面没有响应”
- 失败原因无法直接暴露给用户，增加售后和排查成本

## 处理建议

1. 为组卷失败补充统一错误消息提示
2. 保持生成弹窗关闭、留在当前配置页的原有行为不变
3. 补充页面回归测试，覆盖组卷失败提示分支
