# Issue: 打印下单页初始化加载链路仍存在静默失败

日期：2026-06-22

## 现象

- `frontend-web/src/pages/print/checkout/index.vue` 在加载打印定价和地址列表时仍使用空 `catch`
- 当任一接口失败时，页面会静默回退成空状态，用户无法判断是确实无数据还是初始化异常

## 风险

- 打印下单页属于下单前关键链路，静默失败会直接影响用户是否能完成地址选择和价格确认
- 地址或定价异常被隐藏后，后续订单失败会缺少前置线索，增加排查成本

## 处理建议

1. 为打印定价和地址列表加载失败补充明确提示
2. 保留页面当前兜底行为，避免初始化异常导致渲染中断
3. 增加页面回归测试，覆盖两个初始化请求失败时的提示行为
