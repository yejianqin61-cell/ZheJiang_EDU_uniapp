# Issue: 地址编辑页保存异常仍在使用弱错误类型

日期：2026-06-22

## 现象

- `frontend-web/src/pages/address/edit/index.vue` 的保存分支仍在使用 `catch (error: any)`
- 地址页已经复用共享 `address` API 模块，但异常分支没有同步收口

## 风险

- 地址保存链路的异常对象继续绕开类型系统。
- 后续如果扩展地址校验、默认地址切换或失败提示，弱类型分支会继续扩散。

## 处理建议

1. 使用 `unknown` 接住保存异常
2. 在页面内收口最小错误消息提取逻辑
3. 保持原有保存成功/失败提示与返回行为不变
