# Issue: 管理端定价配置页仍存在静默失败分支

日期：2026-06-22

## 现象

- `frontend-web/src/pages/admin/pricing/index.vue` 在加载定价配置时仍使用空 `catch`
- 保存配置时把弹窗取消和接口失败一并吞掉，真实失败不会向管理员暴露

## 风险

- 定价配置属于核心经营参数，静默失败会导致后台误以为配置已加载或已保存
- 配置变更失败若没有明确提示，后续支付与结算异常会更难回溯

## 处理建议

1. 为定价加载失败补充明确提示
2. 在保存时区分弹窗取消与真实接口失败
3. 增加页面回归测试，覆盖加载失败和保存失败时的提示行为
