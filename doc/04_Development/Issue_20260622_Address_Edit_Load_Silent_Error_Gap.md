# Issue: 地址编辑页加载失败仍存在静默跳回

日期：2026-06-22

## 现象

- `frontend-web/src/pages/address/edit/index.vue` 在编辑模式加载地址详情失败时直接返回上一页
- 页面没有给出明确错误提示，用户只能感知到跳转，无法判断是网络异常、地址不存在还是权限问题

## 风险

- 地址编辑属于下单前的基础信息链路，静默跳回会增加用户困惑和重复操作成本
- 如果详情加载失败没有可见反馈，后续排查地址数据问题会缺少现场线索

## 处理建议

1. 为地址详情加载失败补充明确错误提示
2. 保持现有跳回行为，避免页面停留在不可用状态
3. 增加页面回归测试，覆盖加载失败分支
