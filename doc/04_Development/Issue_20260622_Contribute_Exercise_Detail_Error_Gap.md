# Issue: 练习贡献详情页错误分支反馈不完整

## 问题描述

- `frontend-web/src/pages/contribute/exercise-detail/index.vue` 在详情加载失败时只给出固定“加载失败”，没有透传具体错误
- 删除练习贡献时把确认框取消和真实接口失败混在同一个空 `catch` 里，真实失败会被静默吞掉

## 影响范围

- Web 端练习贡献详情页
- 练习贡献详情加载
- 练习贡献删除操作

## 建议处理

- 为详情加载和删除失败补充统一错误消息提取逻辑
- 区分确认框取消与真实接口失败，仅对真实失败提示用户
- 补充页面回归测试，覆盖详情加载失败和删除失败分支
