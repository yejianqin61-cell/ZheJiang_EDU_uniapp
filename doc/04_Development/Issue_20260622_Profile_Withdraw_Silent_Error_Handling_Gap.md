# Issue: 提现页余额加载链路仍存在静默失败

日期：2026-06-22

## 现象

- `frontend-web/src/pages/profile/withdraw/index.vue` 在初始化余额时仍使用空 `catch`
- 当余额接口失败时，页面会静默回退到 `0`，用户无法判断是接口异常还是账户余额确实为空

## 风险

- 提现页属于资金链路，静默失败会误导用户对可提现额度的判断
- 用户可能在接口异常时继续尝试提现，导致后续提示与页面展示不一致

## 处理建议

1. 为余额加载失败补充明确错误提示
2. 保留默认值兜底，避免页面渲染中断
3. 增加回归测试，覆盖初始化余额加载失败的提示行为
