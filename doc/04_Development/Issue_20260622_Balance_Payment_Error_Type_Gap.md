# Issue: 余额支付与提现页仍在使用弱错误类型

日期：2026-06-22

## 现象

- `frontend-web/src/pages/payment/index.vue` 中创建订单、余额支付、Mock 支付三个分支仍然使用 `catch (error: any)`
- `frontend-web/src/pages/profile/withdraw/index.vue` 中提现提交仍然直接消费 `error?.message`
- 这两页都属于余额资金链路，错误处理一旦继续扩散，会把 `any` 带回支付与提现主流程

## 风险

- 页面错误分支对未知异常对象缺少收敛，类型系统无法约束。
- 资金相关链路里继续保留 `any`，会削弱后续支付与提现回归测试的可维护性。
- 同类错误判断逻辑容易在页面间重复漂移。

## 处理建议

1. 使用 `unknown` 接住页面异常
2. 在页面内收口最小必要的错误识别逻辑
3. 保持原有提示文案和跳转行为不变，只收紧类型与分支判断
