# Issue: 订单列表页模板事件仍依赖弱类型行对象

日期：2026-06-22

## 现象

- `frontend-web/src/pages/orders/index.vue` 的下载订单表格和打印订单表格都在模板里使用 `@row-click="(row: any) => ..."`
- 但页面实际已经通过 `useOrderStore()` 消费共享 `OrderItem[]`
- 这导致订单列表页的行点击事件仍然绕开了共享订单契约

## 风险

- 表格行对象字段一旦变化，模板内的弱类型回调不会被 TypeScript 提前发现。
- 订单主链路仍残留 `any`，容易在后续维护时再次扩散到下载、物流等交互。

## 处理建议

1. 将订单列表行点击事件提升为脚本内显式 typed handler
2. 直接复用共享 `OrderItem` 类型，而不是在模板里写 `any`
3. 保持页面行为不变，只收紧事件签名和标签页类型
