# Issue: 管理端上传页提交异常仍在使用弱错误类型

日期：2026-06-22

## 现象

- `frontend-web/src/pages/admin/upload/index.vue` 的上传提交分支仍使用 `catch (error: any)`
- 页面已经复用共享 `uploadFile()` API，但错误分支没有同步收口

## 风险

- 上传主链路异常对象继续绕开类型系统。
- 上传页后续扩展进度、重试或失败提示时，容易继续扩散弱类型。

## 处理建议

1. 使用 `unknown` 接住上传异常
2. 在页面内收口最小错误消息提取逻辑
3. 保持原有上传进度和成功/失败提示行为不变
