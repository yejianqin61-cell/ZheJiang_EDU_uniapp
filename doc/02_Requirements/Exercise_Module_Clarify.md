## 一、定价（⚠️ 阻塞）

### 1.1 练习模块的定价从哪里来？

现有 pricing_config 表是「下载按题计费、打印分档计费」。但练习模块是完整文件，不存在「题数」。

| 选项 | 说明 |
|------|------|
| A | 全局统一价：管理员在定价配置中设一个「练习试卷单价」，所有练习模块统一 |
| B | 按类目定价：每个单元/专题/考试类目可以单独设置价格 |
| C | 按试卷定价：上传每份试卷时单独填价格 |

**当前设计隐含假设：选项 A。** 请确认。   答复： A

### 1.2 打印服务是否也适用于练习模块？

现有打印服务需要「份数 × 单价」，对完整文件同样适用。但价格是否和 AI 组卷的打印分档一致？还是练习模块独立一套打印定价？

答复：同样适用
### 1.3 如果不支付就能重新抽取，怎么防止薅羊毛？

抽到不喜欢的 → 重新抽 → 还是不支付。每天 3 次限制够不够？是否需要再加一个限制：**每个自然月最多免费抽取 N 次，超出后抽取就需支付**？
答复：不允许重新抽取！

## 二、预览体验（⚠️ 阻塞）

### 2.1 抽取出来的完整文件，预览页展示什么？

AI 组卷的预览页展示前 5 题 + 截断遮罩。但练习模块是完整 PDF/DOCX，没有「题」的概念。

| 选项 | 说明 |
|------|------|
| A | 只显示文件名、页数、大小等元信息，不展示文件内容（防止用户截图白嫖） |
| B | 展示 PDF 第一页的缩略图/截图（后端生成） |
| C | 不设预览，抽取成功后直接显示文件名 + 支付按钮 |
答复：反正不允许重新抽取，那就选择B，只展示第一页缩略图
**当前设计隐含假设：选项 A。** 请确认。

### 2.2 截断策略？

AI 组卷有「前 5 题免费预览，支付后看完整」的截断。练习模块是否需要类似截断？还是完全看不到内容、支付后才能下载？
答复：同2.1，只展示第一页缩略图

## 三、重新抽取（⚠️ 阻塞）：统一答复，一律不允许重新抽取

### 3.1 「每天每类目 3 次」的「类目」是什么粒度？

| 选项 | 示例 |
|------|------|
| A | 四大类型之一：同步练整个模块每天 3 次 |
| B | 具体叶子节点：第 1 课小数乘整数下每天 3 次 |
| C | 全局：所有练习模块合计每天 3 次 |

**当前设计隐含假设：选项 B（具体叶子节点）。** 请确认。

### 3.2 抽了不支付，算一次吗？

算。抽取 = 命中一次配额。

### 3.3 3 次用完后的提示？

```
"今日抽取次数已用完，请明天再来。或使用 AI 组卷功能自主生成试卷。"
```

这样引导用户到 AI 组卷，是否 OK？

### 3.4 如果类目下只有 1 份试卷，还显示「随机抽取」吗？

## 四、抽取范围（❓ 可在开发中定）

## 五、订单与下载（❓ 可在开发中定）

### 5.1 订单 type 新增 'exercise'，和 'download' 有什么区别？

两者最终都是下载文件。区别在于：
- `download`：AI 组卷 → 需要后端生成 DOCX/PDF → 导出
- `exercise`：练习模块 → 文件已经在 COS 上 → 直接返回下载 URL，不需要导出服务

是否需要区分？还是统一用 `download` type，后端根据来源做不同处理？
答复：同意

### 5.2 支付后重复下载？

和现有逻辑一致：已支付的练习订单也支持在历史订单中重复下载。
答复：可以

### 5.3 文件格式转换？

管理员上传 DOCX，用户选 PDF 导出 → 是否需要 LibreOffice 转换？还是用户只能下载原始格式？
答复：只能下载原格式
