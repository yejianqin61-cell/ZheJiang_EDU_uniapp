## Q1：试卷列表在哪展示？

当前流程：类目页 → 点「AI抽取」→ draw 页（随机抽一个）

新流程需要决定试卷列表在哪展示：

**方案 A**：类目页内嵌展示

```
类目页
├── 五年级数学上册（单元）
│   ├── 第一章 小数乘法  [查看试卷 ▼]
│   │   ├── 📄 小数乘法单元测试卷  DOCX 45KB
│   │   ├── 📄 小数乘法提高卷    PDF  120KB
│   │   └── ...
│   └── 第二章 位置      [查看试卷 ▼]
└── ...
```

- 优点：少一次页面跳转，信息集中
- 缺点：类目页可能很长，需要折叠/展开

**方案 B**：点击类目 → 跳转新页面展示试卷列表

```
类目页 → 点击类目 → /exercises/papers?categoryId=xxx
┌──────────────────────────────────┐
│ 五年级数学上册 · 第一章 小数乘法      │
│                                  │
│ ┌────┬──────┬──────┬──────┐     │
│ │缩略│试卷标题 │DOCX  │ 45KB │     │
│ │图  │        │      │      │     │
│ ├────┼──────┼──────┼──────┤     │
│ │缩略│试卷标题 │PDF   │120KB │     │
│ │图  │        │      │      │     │
│ └────┴──────┴──────┴──────┘     │
└──────────────────────────────────┘
```

- 优点：页面干净，可以和现有 draw 页复用
- 缺点：多一次点击

> **倾向**：方案 B。类目页已经很长了，再加内嵌列表体验差。draw.vue 改造成试卷列表页。

**你的选择**：A / B？
答复：B

## Q2：收费节点在哪？

当前流程：随机抽题 → 看到试卷（已付费标记）→ 下载/打印另付费

**方案 A**：浏览免费，下载/打印才收费
- 类目页 → 试卷列表（免费）→ 点击试卷 → 预览详情（免费）→ 下载/打印 → 创建订单

**方案 B**：查看试卷就收费（和现在一样）
- 类目页 → 试卷列表（免费）→ 点击试卷 → 创建订单 → 支付后查看详情

> **倾向**：方案 A。浏览免费降低门槛，下载/打印才收费。和"下载服务"的定位一致。

**你的选择**：A / B？
答复：A

## Q3：缩略图在列表中显示吗？

**方案 A**：显示缩略图（小图），试卷卡片含缩略图 + 标题 + 文件信息
**方案 B**：纯文本列表，点击后才看到缩略图

> **倾向**：方案 A。视觉吸引力更强，教师可以凭缩略图判断试卷内容。

**你的选择**：A / B？
答复：b
---

## Q4：同步练的「课时」怎么处理？

同步练的层级是：单元 → 课时 → 试卷。当前每个课时一个 draw 按钮。

**方案 A**：单元和课时都展示试卷。点单元展开课时列表，每个课时下展示试卷。
**方案 B**：只在课时级别展示试卷，单元级别只用来展开课时。

> **倾向**：方案 A。和单元练/专题练体验统一。

**你的选择**：A / B？
答复:A
---

## Q5：如果没有试卷，显示什么？

某个类目/课时下可能没有任何试卷（还没上传）。

**方案 A**：显示空状态「暂无试卷」，引导教师去贡献页上传
**方案 B**：直接隐藏该类目/课时

> **倾向**：方案 A。让用户知道这个类目存在但没有内容，也许他们愿意上传。

**你的选择**：A / B？
答复:A,但是我们只允许在已存在的类目下上传文件。所以如果当前类目没有被创建，那就不展示引导上传的按钮

## Q6：旧的 draw_record 表要不要删？

这个表用于防止用户重复抽题。改为列表浏览后不再需要。

**方案 A**：删除表和相关代码（清理干净）
**方案 B**：保留但不再使用（怕以后要恢复）

> **倾向**：方案 A。

**你的选择**：A / B？
答复B
